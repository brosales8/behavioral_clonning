Project Description:¶

The goals / steps of this project are the following:

  • Use the simulator to collect data of good driving behavior
  • Build, a convolution neural network in Keras that predicts steering angles from images
  • Train and validate the model with a training and validation data set
  • Test that the model successfully drives around track one without leaving the road
  • Summarize the results with a written report

Pipeline Description¶

The pipeline proposed to the solution consisted of 4 steps:

  1. Data Collection
  2. Preprocessing and Data Augmentation
  3. Model Architecture and Training
  4. Testing the model in the Udacity Simulators

Step 1: Data Collection, Analysis and Visualization¶

The data was collected using the Udacity windows version simluator. It was set in the lowest resolution '320x240' and the graphics quality 'Fastest'. The data collection process was the most dificult part of this project and therefore, I realized the importance of gathering quality data make a big difference in the results. In the beginning, I collected a couple laps in track1 using both directions and tried to fit the model with that data. However, the model had a high error (> 4.0) in both set (training and validation) and the car got outside the track. Then, my second approach was to put more focus in the data distribution and therefore I analyzed the data using a histogram and filtered curves and straight. For this I used a threshold of 0.15. Then, angles greater than 0.15 and less than -0.15 were called curves, the zero angles were named zeros, and the rest angles between -0.15 and 0.15 exclusive were called straight. Next, I generated more data based on the number of curves and straight and chose the amount of angles for each category (zeros, straight, left and right curves) looking for a uniform distribution. Eventhough the distribution was not completely uniform, the data was enoughly good to train the algorithm. The next table present the summary of the data distribution to the center camera:

Step 2: Preprocessing and Data Augmentation¶

The strategy to data augmentation basically was:

1. Cameras: I used the 2 additional images provided by left and right cameras and adding a correction factor of 0.06 to those angles. The number of images availables to train and validate the model increased from 13,283 to 39,849. Using those images left and right was not neccesary to create recovery picture.

2. Brightness: The images brightness were changed randomly with a 0.5 probability. For that, the image was converted to HSV format then the layer 'v' was multiply by a random factor between 0.5 and 1. This method produced images with different brightness which help the model to generalize.

3. Flipping: Only to the center images with absolute angles greater than 0.4 was applied flipping method. The idea was to generate new images since center camera avoiding the model to be biased.

4. RGB format: After several experiments, the model did show a significant improvement using YUV format just like the NVIDIA mentioned in its paper. Therefore, RGB was chosen to facilitate the visualization.

Step 3: Model Architecture and Training¶

Setup Hyperparameters¶

  • EPOCHS = 3
  • BATCH_SIZE = 150
  • Optimizer = Adam

The chosen architectura was based on Nvidia paper 'End to End Learning for Self-Driving Cars'. The architecture description is:

  • 5 Convolutional Layers
  • 1 Flatten Layer
  • 4 Fully Connected
  • The python package used was keras

The convulotional layers help the model in the feature extraction process and the fully connected layers works as a steering controller. This model is suitable to solve this problem because the big challenge is to learn the lines lane of the track and mantains the car on the center. The 6 convolutional layers are able to extract the features of the lines lane using the filters and depth of each layer. Also, the track 1 do not present aggressive changes in the images captured by the cameras. Another important fact is that a similar architectura has been used in real self-driving cars with excellent results, therefore the network is already probed to work.

Here the description for each layer:

model = Sequential()

Input¶

The Project 3 architecture accepts a 160x320x3 images as input, where 3 indicates that the image is in color, 160 pixels height and 320 pixels width.

Preprocessing Layer¶

  • The images were cropped to capture the interested zone on which the model should learn to predict the angle. Then a cropping function is applyed and as a result the output is a image with shape (65,320,3). Also, the cropping helps the algorithm to be faster because the image contains less pixeles.

    model.add(Cropping2D(cropping=((70,25), (0,0)), input_shape=(160,320,3)))

  • The second part is to normalize the image setting a mean 0 and range between -0.5 to 0.5.

    model.add(Lambda(lambda x: x / 255.0 - 0.5))

Layer 1 to 3:¶

Three convolutional layers with a kernel=5x5, Stride=2x2, Activation='Relu', Padding='Valid', regularization l2 in W matrix. The depth was incremented in each convolution starting with 24, 36 and finally 48. The Relu function help the model avoid linearity.

model.add(Conv2D(24,5,strides=2,activation='relu', padding='valid', kernel_regularizer = l2(0.001)))
model.add(Conv2D(36,5,strides=2,activation='relu', padding='valid', kernel_regularizer = l2(0.001)))
model.add(Conv2D(48,5,strides=2,activation='relu', padding='valid', kernel_regularizer = l2(0.001)))

Layer 4 to 5:¶

Two convolution layers with a kernel=3x3, Activation='Relu', Padding='Valid', regularization l2 in W matrix. For this two convolution, the depth is 64.

model.add(Conv2D(64,3,activation='relu', padding='valid', kernel_regularizer = l2(0.001)))
model.add(Conv2D(64,3,activation='relu', padding='valid', kernel_regularizer = l2(0.001)))

Layer 6:¶

One flatten layer model.add(Flatten())

Layer 7 to 9:¶

Four fully connected layers with outputs 100, 50, 10, and three dropping layers to help avoiding overfitting. model.add(Dropout(0.5)) model.add(Dense(100,kernel_regularizer = l2(0.001))) model.add(Dropout(0.5)) model.add(Dense(50,kernel_regularizer = l2(0.001))) model.add(Dropout(0.5)) model.add(Dense(10,kernel_regularizer = l2(0.001))))

Output¶

One fully connected with output 1, where 1 indicates the steering angle calculated by the network model.add(Dense(1,kernel_regularizer = l2(0.001)))

The optimizer used was Adam therefore the learning rate was not tuned manually. The loss was measured using 'mse' because of the output to predict is a numeric value.

Also, the keras fit_generator was used in order to processing batch size without having the total images loaded in RAM. The generator improved the performance of the algorithm signicantly.

model.compile(loss='mse', optimizer= 'adam')

history_object = model.fit_generator(trainingGen, steps_per_epoch=math.ceil(len(tPaths)/batchSize), epochs=3, verbose=1,

validation_data=validGen, validation_steps=math.ceil(len(vPaths)/batchSize), initial_epoch=0)

Training and Testing Results¶

During several attempts to reduce overfitting and underfitting, I tried l2 generalization and dropping layers. However, I did not see for sustantial improvement. Instead, I was more on focus in the quality of the data and tunning basically 2 parameteres correction (0.06) factor for lateral cameras and flipping angles selected.

In the begining, the car was driving out of the track on the curves, but capturing more specialized data on the curves, I was able to correct the issue. Next, I was focus on correct the oscillation on the track. For that, I tunned the previous parameters: angles correction on sides cameras and the parameter to flip the image. As a consequence, I achieved to reduce the oscillations on track and improve the way how the car drove on the curves.

The model was trained and validated on different data sets. I splited the data using 80% for training and 20% for validation. The model was tested by running it through the simulator and ensuring that the vehicle could stay on the track.

  • Training Time: 1052 seconds (17.5 Minutes)
  • Training loss: 0.0046
  • Validation loss: 0.0044

The losses in both sets look to be low. The results do not present underfitting or overfitting symptoms.
